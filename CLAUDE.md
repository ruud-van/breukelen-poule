# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Setup

```bash
python3 -m venv .venv
.venv/bin/pip install xlrd numpy scipy
```

## Build & run

```bash
# Fetch completed results from ESPN -> results_state.json
.venv/bin/python results_fetch.py

# Production build: Excel + wk_odds.json + results_state.json -> odds.json + data.json
.venv/bin/python build_pool.py

# Stamp data.json into the HTML template -> index.html
.venv/bin/python build_page.py

# IMPORTANT: after every local build, discard the odds.json diff (it is intentionally never committed)
git checkout odds.json

# View locally (or use the http.server launch config on port 8743)
open index.html
```

Verify embedded JS after a template change in `build_page.py`:
```bash
node --check <(sed -n '/<script>/,/<\/script>/p' index.html | sed '1d;$d')
```

## Architecture

The pipeline is: `results_fetch.py` → `build_pool.py` → `build_page.py` → `index.html`.

**Data flow**

| File | Role |
|---|---|
| `WK_2026__allen_.xls` | Master Excel with 58 participants and their 72 predictions (committed to repo) |
| `wk_odds.json` | Raw 1X2 + over/under snapshot from The Odds API — one-time frozen input, never re-fetched |
| `results_state.json` | Completed match results from ESPN; `_updated_at` only refreshes when results change (prevents empty bot commits) |
| `odds.json` | Frozen correct-score odds derived by `build_pool.py` — **never commit**, always `git checkout odds.json` after a local build |
| `data.json` | ~280 KB: participants, standings, streaks, predictions (58×72), similarity network — the page's data source |
| `index.html` | Static single-file page; fully generated by `build_page.py` from its embedded HTML/CSS/JS template |
| `regels.html` | Static standalone page with the new WK 2026 rules (FIFA/IFAB); also generated by `build_page.py`, linked from the Stand tab |

**Key modules**

- `parse_form.py` — reads one `.xls` participant form; contains `TEAMS_NL_EN` (the canonical Dutch→English team name table used everywhere)
- `parse_all.py` — calls `parse_form.py` for all participants in the master Excel
- `odds_model.py` — bivariate Poisson + Dixon-Coles; converts 1X2 + over/under consensus odds into a correct-score grid with configurable overround
- `odds_fetch.py` — one-time fetch from The Odds API; also exports `to_nl(name)` which maps English/ESPN team names back to Dutch names (used by both `results_fetch.py` and `build_pool.py`)
- `results_fetch.py` — queries the undocumented ESPN scoreboard endpoint (no key required); filters on `season.slug == "group-stage"` and `status.type.completed`
- `build_pool.py` — production build; constants `START=2000`, `STAKE_TOTO=100`, `STAKE_SCORE=20` at the top; also runs a Monte Carlo budget simulation (for info only, not written to the page)
- `build_page.py` — contains the full HTML/CSS/JS template inline; renders 3 tabs: Stand (standings + budget chart), Voorspellingen (predictions accordion), Gelijkenis (MDS similarity network)

**GitHub Actions**

`.github/workflows/update-pool.yml` runs 5×/day (cron `10 0,3,6,13,21 * * *` UTC). It only commits `data.json`, `index.html`, `regels.html`, and `results_state.json` — never `odds.json`. The deployed page is at https://ruud-van.github.io/breukelen-poule/.

**Naming / translation chain**

Dutch names (forms + `odds.json`) ↔ English names (The Odds API / ESPN) via `to_nl()` in `odds_fetch.py`. ESPN uses non-standard spellings (`Czechia`, `Bosnia-Herzegovina`, `United States`) handled by `FEED_ALIASES` in that same file.
